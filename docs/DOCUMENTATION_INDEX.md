## Terminology

Understanding Beads-specific terms:

| Term | Definition | Learn More |
|------|------------|------------|
| **Issue** | A tracked work item (bug, feature, task, epic, chore) | [USER_GUIDE.md](USER_GUIDE.md#issue-types) |
| **Hash ID** | Collision-resistant issue identifier (e.g., `bd-a1b2`) | [USER_GUIDE.md](USER_GUIDE.md#issue-ids-hash-based) |
| **Dependency** | Relationship between issues (blocks, related, parent-child, discovered-from) | [USER_GUIDE.md](USER_GUIDE.md#dependency-types) |
| **Ready work** | Issues with no open blockers | [USER_GUIDE.md](USER_GUIDE.md#finding-work) |
| **JSONL** | JSON Lines format for storing issues in Git | [ADVANCED.md](ADVANCED.md) |
| **Sync** | Export/import/commit/push operation | [USER_GUIDE.md](USER_GUIDE.md#sync-with-git) |
| **Daemon** | Background process for auto-sync | [DAEMON.md](DAEMON.md) |
| **Epic** | Large issue with hierarchical children (e.g., `bd-a1b2.1`, `bd-a1b2.2`) | [USER_GUIDE.md](USER_GUIDE.md#hierarchical-issues-epics) |
| **Compaction** | Memory decay for old issues | [USER_GUIDE.md](USER_GUIDE.md), [ADVANCED.md](ADVANCED.md) |
| **Agent Mail** | Real-time multi-agent coordination (optional) | [AGENT_MAIL.md](AGENT_MAIL.md) |
| **MCP** | Model Context Protocol for AI integration | [../integrations/beads-mcp/README.md](../integrations/beads-mcp/README.md) |
| **Stealth mode** | Local-only Beads usage (no shared files) | [USER_GUIDE.md](USER_GUIDE.md#stealth-mode-personal-use-only) |

## Documentation Conventions

Throughout the documentation, we use these conventions:

### Command Examples

```bash
# Comments explain what the command does
bd create "Issue title" --description "Details"
```

### File Paths

- **Absolute paths:** When referring to files in the repository (e.g., `docs/USER_GUIDE.md`)
- **Relative paths:** When linking between docs (e.g., `[INSTALLING.md](INSTALLING.md)`)

### Cross-References

- **See [DOCUMENT.md](DOCUMENT.md)** - Links to another document
- **See [DOCUMENT.md](DOCUMENT.md#section)** - Links to specific section
- **For more details...** - Indicates optional reading for deeper understanding

## Contributing to Documentation

Documentation improvements are always welcome!

**To contribute:**
1. Read [DEVELOPER_GUIDE.md](DEVELOPER_GUIDE.md) for setup
2. Make changes to relevant .md files
3. Test any command examples
4. Update this index if adding new docs
5. Submit a pull request

**Documentation style:**
- Clear and concise
- Include examples
- Specify audience
- Cross-link related docs
- Use consistent formatting
